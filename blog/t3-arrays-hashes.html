Arrays and Hashes

By: Arthur Head on June 30, 2015

When writing a program, a programmer must decide how to structure the data in the program. Two available options are arrays and hashes.

Arrays are ordered, integer-indexed collections of any object. The index of an array starts at 0 and not 1 like you would naturally think. Arrays do have a length property though, and it actual represents the length of the array starting at 1. Arrays can consist of any object, so that means you could have an array of strings, numbers, or even an array of arrays. The proper way to declare an array is to assign it a variable name and surround the actual array in square brackets like this: my_array = [1,2,3,4,"arthur","head",5,6,7,8]. In this example the first element in the array, the number 1, is actually at index 0. To access an element in an array you just simply reference it's index like this: my_array[0] which would return the number 1. To access each element in the array you can use iteration. Iterating over an array is a nifty way to access each element without having to manually reference each index.

A Hash is a dictionary-like collection of unique keys and values. Hashes are not ordered by integer index like Arrays, and instead use a key that can consist of any object to reference a particular value. Hashes can be created a couple different ways. You can use the new method like this: new_hash = Hash.new. The .new is the built in Hash method that instatiates a new Hash assigned to the variable name new_hash. A hash can also be created using the implicit form which looks a bit different: newer_hash = {"first_name" => "Arthur", "last_name" => "Arthur"}. To access a value in a hash, simply reference the key like this: new_hash["first_name"].
