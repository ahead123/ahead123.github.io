Ruby Classes

By: Arthur Head on July 20, 2015

Ruby provides a way to group many methods into one happy family. The way to do this is by creating a Class. A Ruby Class is simply a way to bundle behaviors(methods) and create objects that can then share these behaviors. A common method in a class is an initialize method. The initalize method is used to set the initial action that takes place when an instance of a Class is created or initialized. In the example above, the initialize method takes two parameters upon inititialization: name, and age. If we tried to create an instance of this class without providing a name and age, we would get an error.

Classes also contain instance variables. Instance variables are identifiable by the @ sign that precedes them. Instance varaiables provide a way to store values between each instance of a class. So everytime we create a new Person the different values, and ages for each Person will be stored in the @name, and @age instance varaibles. So this gives us a way to keep track of many people and many ages, without overwriting the values. The attr_accessor above is a Ruby Module that allows us to access (read, and change) the values of age and name. So after we create a Person, we can later go back and update the person's age and name if need be, using the attr_accessor module.

First we create a new instance of the Person Class by writing Person.new and passing in the required name and age parameters. We also set this instance equal to a variable so we can recall it later

Then we call the method introcude on our new person. Since it belongs to the Person Class it can use the method introduce.

The we call the other method in our Person Class, meet_someone_new. Again since this object belongs to the person class it can access and use all the methods in the Class.
